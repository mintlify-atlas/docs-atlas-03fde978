ROCm supports two main AMD GPU architecture families: **CDNA** (Compute DNA), targeting HPC and AI workloads on AMD Instinct accelerators, and **RDNA** (Radeon DNA), targeting graphics and compute on Radeon GPUs. Each generation improves compute density, memory bandwidth, and support for new data types.

## CDNA architecture (Instinct series)

<Tabs>
  <Tab title="CDNA4 — MI350 / MI355X">
    The CDNA4 architecture (LLVM target `gfx950`) powers the AMD Instinct MI350X and MI355X accelerators. It continues the multi-XCD chiplet design introduced with CDNA3, with 8 XCDs per package providing up to 256 active Compute Units (32 per XCD).

    **Key improvements over CDNA3:**
    - Up to 288 GiB of HBM3 memory
    - 256 MiB L3 cache
    - 160 KiB Local Data Share (LDS) per CU
    - Enhanced Matrix Core support for AI inference data types

    | Spec | MI355X / MI350X |
    |------|----------------|
    | Architecture | CDNA4 |
    | LLVM target | gfx950 |
    | VRAM | 288 GiB HBM3 |
    | Compute Units | 256 (32 per XCD) |
    | Wavefront size | 64 |
    | LDS | 160 KiB |
    | L3 cache | 256 MiB |
    | L2 cache | 32 MiB (4 per XCD) |
    | VGPR file | 512 KiB |
  </Tab>
  <Tab title="CDNA3 — MI300 series">
    The CDNA3 architecture (LLVM target `gfx942`) powers the AMD Instinct MI300X, MI300A, and MI325X. It introduces the **Accelerated Complex Die (XCD)** as the fundamental GPU compute building block.

    Each XCD contains 40 Compute Units (38 active, 2 reserved for yield management), four Asynchronous Compute Engines (ACEs), and a 4 MiB L2 cache. The MI300X integrates 8 XCDs, 4 I/O dies, and 8 stacks of HBM3 via AMD Infinity Fabric, providing up to 304 active CUs.

    The MI300A is an Accelerated Processing Unit (APU) that combines CPU cores (AMD EPYC) with 6 XCDs (228 active CUs) and 128 GiB of unified HBM3 memory shared between CPU and GPU.

    **CDNA3 Matrix Core improvements over CDNA2:**
    - 3× throughput for FP16 and BF16
    - 6.8× throughput for INT8
    - 16× throughput for FP8 vs FP32
    - New TF32 support with 4× throughput vs FP32

    | LLVM target | gfx942 | gfx942 | gfx942 |

    | LDS | 64 KiB | 64 KiB | 64 KiB |
    | L3 cache | 256 MiB | 256 MiB | 256 MiB |

    | Data type | FLOPS/clock/CU | Peak TFLOPS |
    |-----------|---------------|-------------|
    | Matrix FP64 | 256 | 163.4 |
    | Vector FP64 | 128 | 81.7 |
    | Matrix FP32 | 256 | 163.4 |

    The MI300X OAMs attach to the host system via PCIe Gen 5 x16 links. In an 8-GPU node, seven AMD Infinity Fabric links per GPU form a fully connected GPU fabric.
  </Tab>
  <Tab title="CDNA2 — MI250 series">
    The CDNA2 architecture (LLVM target `gfx90a`) powers the AMD Instinct MI250X, MI250, and MI210. It uses a **Graphics Compute Die (GCD)** structure: the MI250X and MI250 are dual-GCD packages, each GCD presenting as a separate GPU device.

    Each GCD has 104 active CUs (MI250), organized into four SIMD units per CU. Each SIMD unit processes 16 FP64 elements per instruction, so a full wavefront of 64 elements occupies all four SIMDs simultaneously. The MI250 GCD peaks at 22.6 TFLOPS (vector FP64) at 1.7 GHz clock.

    Memory uses HBM2e: 1.6 TB/s peak bandwidth per GCD, 3.2 TB/s total for the dual-GCD MI250X OAM.

    |------|--------|-------|-------|

    | Wavefront size | 64 | 64 | 64 |
    | L2 cache | 16 MiB | 16 MiB | 8 MiB |

    **MI250 OAM peak performance (both GCDs):**

    | Data type | FLOPS/clock/CU | Peak TFLOPS |
    |-----------|---------------|-------------|
    | Matrix FP64 | 256 | 90.5 |
    | Vector FP64 | 128 | 45.3 |
    | Matrix FP32 | 256 | 90.5 |
    | Matrix FP16 | 1024 | 362.1 |
    | Matrix BF16 | 1024 | 362.1 |
    | Matrix INT8 | 1024 | 362.1 |

    The GCDs within an OAM connect via four AMD Infinity Fabric links at 25 GT/s, giving 200 GB/s peak transfer rate between GCDs (400 GB/s bidirectional).
  </Tab>
  <Tab title="CDNA1 — MI100">
    The CDNA1 architecture (LLVM target `gfx908`) is the first AMD Instinct generation to separate graphics from compute, focusing exclusively on HPC and AI workloads.

    The MI100 has 120 Compute Units organized into 8 Shader Engines (15 CUs each). Each CU contains four SIMD units that process 16-element SIMD instructions, enabling 64-wide wavefront execution. Four stacks of HBM2 provide 32 GiB of memory with a 4,096-bit memory interface at 1.228 TB/s peak bandwidth.

    Each CU holds up to 10 concurrent wavefronts. The CU allocates 256 VGPRs and 800 SGPRs dynamically. VGPR usage directly affects wavefront occupancy — for example, using 119 VGPRs limits the CU to 2 concurrent wavefronts.

    **MI100 peak performance:**

    | Data type | FLOPS/clock/CU | Peak TFLOPS |
    |-----------|---------------|-------------|
    | Vector FP64 | 64 | 11.5 |
    | Matrix FP32 | 256 | 46.1 |
    | Vector FP32 | 128 | 23.1 |
    | Matrix FP16 | 1024 | 184.6 |
    | Matrix BF16 | 512 | 92.3 |

    The MI100 connects to host processors via PCIe Gen 4 x16 links (16 GT/s). Up to four GPUs in a node can form a fully connected fabric using AMD Infinity Fabric bridge cards at 23 GT/s.
  </Tab>

## Key architectural concepts

### Compute Units (CUs)

A Compute Unit is the fundamental execution building block of AMD GPUs. Each CU contains:

- **Four SIMD units**, each capable of processing 16 data elements per instruction
- **Vector General Purpose Registers (VGPRs)**: hold per-lane data for the 64 lanes of a wavefront
- **Scalar General Purpose Registers (SGPRs)**: hold data common to all lanes in a wavefront
- **Local Data Share (LDS)**: fast on-chip shared memory accessible by all threads in a workgroup
- **L1 instruction cache** and **L1 scalar cache** shared across CU pairs
- **L1 vector cache** (also called L0 on RDNA) per CU pair

### Wavefronts

A **wavefront** (analogous to a CUDA warp) is the fundamental unit of execution on AMD GPUs. A wavefront is a group of 64 threads (or 32 on RDNA) that execute the same instruction in lockstep across the four SIMD units of a CU.

- On CDNA hardware, wavefront size is always **64**
- On RDNA hardware, wavefront size is **32 or 64** (determined at dispatch)
- Multiple wavefronts can be in flight on a single CU simultaneously (occupancy), hiding memory latency through context switching

### SIMD units

Each CU has four SIMD-16 units. For a 64-wide wavefront:

- Each SIMD unit handles 16 lanes
- All four SIMDs execute the same instruction in the same clock cycle
- For FP64 double-precision operations, this yields 64 FP64 results per clock per CU

### Memory hierarchy

AMD GPU memory is organized in a hierarchy of decreasing speed and increasing size:

| Level | Type | Scope | Typical size |
|-------|------|-------|-------------|
| Registers (VGPR/SGPR) | On-chip | Per wavefront | 512 KiB (CDNA3) |
| LDS | On-chip | Per workgroup | 64–160 KiB per CU |
| L1 cache | On-chip | Per CU or CU pair | 16–32 KiB |
| L2 cache | On-chip | Per GCD/XCD | 4–8 MiB |
| L3 / Infinity Cache | On-chip | Per package | 256 MiB (CDNA3/4) |
| HBM / GDDR | Off-chip | Global | 16–288 GiB |

**High Bandwidth Memory (HBM)** is used on all Instinct MI-series GPUs. HBM stacks are placed adjacent to the GPU die using 2.5D packaging, providing extremely wide memory buses (4,096 bits on MI100) with peak bandwidths of 1.2–5.3 TB/s depending on generation.

**Local Data Share (LDS)** is a fast scratchpad memory shared among all threads in a workgroup. It is software-managed and must be explicitly used in kernel code. Efficient LDS usage is critical for memory-bound kernels.
