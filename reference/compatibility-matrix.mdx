Use this matrix to view ROCm compatibility and system requirements across successive major and minor releases. The table covers GPU architectures, Linux distributions, ML frameworks, communication libraries, and component versions.

<Note>
This page reflects ROCm versions 7.2.1, 7.2.0, and 6.4.0. For the most current information on supported operating systems and GPUs, see the [ROCm Install on Linux system requirements](https://rocm.docs.amd.com/projects/install-on-linux/en/latest/reference/system-requirements.html).
</Note>

<Warning>
GPUs listed in this matrix support compute workloads only (no display or graphics). If you're using ROCm with AMD Radeon GPUs or Ryzen APUs for graphics workloads, see [Use ROCm on Radeon and Ryzen](https://rocm.docs.amd.com/projects/radeon/en/latest/) to verify compatibility.
</Warning>

For AMD Instinct MI325X KVM SR-IOV users, do not use AMD GPU Driver (amdgpu) version 30.20.0. Starting from ROCm 6.4.0, forward and backward compatibility between the AMD GPU Driver and its user-space software is provided up to a year apart.
