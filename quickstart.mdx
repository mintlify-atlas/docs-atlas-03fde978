This guide walks you through installing ROCm on Linux, verifying your GPU is detected, and running a simple HIP program.

## Prerequisites

### Supported operating systems

ROCm supports the following Linux distributions:

| Distribution | Versions |
|---|---|
| Ubuntu | 22.04, 24.04 |
| Red Hat Enterprise Linux (RHEL) | 8.x, 9.x |
| SUSE Linux Enterprise Server (SLES) | 15 SP5, 15 SP6 |
| Oracle Linux | 8.x, 9.x |
| Debian | 12 |
| Rocky Linux | 8.x, 9.x |

### Hardware requirements

ROCm supports AMD CDNA (Instinct) and RDNA (Radeon) GPU architectures. Check the [compatibility matrix](/reference/compatibility-matrix) to confirm your GPU is supported for your target ROCm version.

<Warning>
  ROCm requires a supported GPU with PCIe atomics enabled. Ensure your system BIOS has IOMMU/SR-IOV or PCIe atomics enabled. Virtual machines may require additional configuration.
</Warning>

### System requirements

- 64-bit x86_64 processor
- At least 16 GB of system RAM (32 GB+ recommended for AI workloads)
- AMD GPU with ROCm-compatible firmware
- Kernel version supported by your ROCm release (see the [compatibility matrix](/reference/compatibility-matrix))

## Installation

<Steps>

<Step title="Add the AMD ROCm repository">

Add the AMD GPG key and package repository for your Linux distribution.

<Tabs>
  <Tab title="Ubuntu / Debian">
    ```bash
    # Install prerequisites
    sudo apt update
    sudo apt install -y wget gnupg

    # Add the AMD ROCm repository key
    wget https://repo.radeon.com/rocm/rocm.gpg.key -O - | \
      gpg --dearmor | \
      sudo tee /etc/apt/keyrings/rocm.gpg > /dev/null

    # Add the ROCm repository (replace <version> with target, e.g. 6.3)
    echo "deb [arch=amd64 signed-by=/etc/apt/keyrings/rocm.gpg] \
      https://repo.radeon.com/rocm/apt/<version> jammy main" | \
      sudo tee /etc/apt/sources.list.d/rocm.list

    sudo apt update
    ```
  </Tab>
  <Tab title="RHEL / Rocky / Oracle Linux">
    ```bash
    # Add the ROCm repository (replace <version> with target, e.g. 6.3)
    sudo tee /etc/yum.repos.d/rocm.repo <<EOF
    [ROCm]
    name=ROCm
    baseurl=https://repo.radeon.com/rocm/rhel9/<version>/main
    enabled=1
    gpgcheck=1
    gpgkey=https://repo.radeon.com/rocm/rocm.gpg.key
    EOF

    sudo dnf clean all
    ```
  </Tab>
  <Tab title="SLES">
    ```bash
    # Add the ROCm repository (replace <version> with target, e.g. 6.3)
    sudo zypper addrepo \
      https://repo.radeon.com/rocm/zyp/<version>/main rocm

    sudo zypper --gpg-auto-import-keys refresh
    ```
  </Tab>
</Tabs>

</Step>

<Step title="Install ROCm packages">

Install the `rocm` meta-package, which pulls in the full ROCm stack including the runtime, HIP, libraries, and tools.

<Tabs>
  <Tab title="Ubuntu / Debian">
    ```bash
    sudo apt install -y rocm
    ```
  </Tab>
  <Tab title="RHEL / Rocky / Oracle Linux">
    ```bash
    sudo dnf install rocm
    ```
  </Tab>
  <Tab title="SLES">
    ```bash
    sudo zypper install rocm
    ```
  </Tab>
</Tabs>

<Tip>
  To install a specific ROCm version, use a versioned package name such as `rocm-6.3.0` instead of `rocm`.
</Tip>

</Step>

<Step title="Configure your user and environment">

Add your user to the `render` and `video` groups so you can access GPU devices without root privileges.

```bash
sudo usermod -aG render,video $USER
```

Add the ROCm binary paths to your shell environment:

```bash
echo 'export PATH=$PATH:/opt/rocm/bin' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/opt/rocm/lib' >> ~/.bashrc
source ~/.bashrc
```

<Warning>
  Log out and back in (or reboot) for group membership changes to take effect.
</Warning>

</Step>

<Step title="Verify the installation">

Run `rocminfo` to list detected AMD GPU agents and confirm the ROCm runtime is working:

```bash
rocminfo
```

You should see output similar to:

```text

...
*******
Agent 1
*******
  Name:                    AMD Instinct MI300X
  Uuid:                    GPU-XXXXXXXXXXXXXXXX
  Marketing Name:          AMD Instinct MI300X
  Vendor Name:             AMD
  Feature:                 KERNEL_DISPATCH
  Profile:                 FULL_PROFILE
  Float Round Mode:        NEAR
  Max Queue Number:        128(0x80)
  Queue Min Size:          64(0x40)
  Queue Max Size:          131072(0x20000)
  ...
```

Run `rocm-smi` to check GPU utilization, memory, temperature, and power draw:

```bash
rocm-smi
```

Expected output:

```text
========================================== ROCm System Management Interface ==========================================
====================================================== Info ======================================================
GPU  Temp (DieEdge)  AvgPwr  SCLK    MCLK     Fan     Perf  PwrCap  VRAM%  GPU%
  0  35.0c           23.0W   800Mhz  900Mhz   0%      auto  750.0W    0%   0%

```

If your GPU appears in both outputs, ROCm is installed and working correctly.

</Step>

## Your first HIP program

HIP (Heterogeneous-Compute Interface for Portability) is the primary programming model for ROCm. The following example adds two vectors on the GPU.

### Vector addition in HIP

Create a file named `vector_add.cpp`:

```cpp
#include <hip/hip_runtime.h>
#include <stdio.h>
#include <stdlib.h>

#define N 1024

// GPU kernel: add two vectors element-wise
__global__ void vectorAdd(const float* a, const float* b, float* c, int n) {
    int i = blockDim.x * blockIdx.x + threadIdx.x;
    if (i < n) {
        c[i] = a[i] + b[i];
    }
}

int main() {
    size_t bytes = N * sizeof(float);

    // Allocate host memory
    float* h_a = (float*)malloc(bytes);
    float* h_b = (float*)malloc(bytes);
    float* h_c = (float*)malloc(bytes);

    // Initialize input vectors
    for (int i = 0; i < N; i++) {
        h_a[i] = (float)i;
        h_b[i] = (float)(N - i);
    }

    // Allocate device memory
    float *d_a, *d_b, *d_c;
    hipMalloc(&d_a, bytes);
    hipMalloc(&d_b, bytes);
    hipMalloc(&d_c, bytes);

    // Copy input data to device
    hipMemcpy(d_a, h_a, bytes, hipMemcpyHostToDevice);
    hipMemcpy(d_b, h_b, bytes, hipMemcpyHostToDevice);

    // Launch kernel: 256 threads per block
    int threadsPerBlock = 256;
    int blocksPerGrid = (N + threadsPerBlock - 1) / threadsPerBlock;
    hipLaunchKernelGGL(vectorAdd, blocksPerGrid, threadsPerBlock, 0, 0,
                       d_a, d_b, d_c, N);

    // Copy result back to host
    hipMemcpy(h_c, d_c, bytes, hipMemcpyDeviceToHost);

    // Verify result
    int errors = 0;
    for (int i = 0; i < N; i++) {
        float expected = h_a[i] + h_b[i];
        if (h_c[i] != expected) {
            printf("Mismatch at index %d: got %f, expected %f\n", i, h_c[i], expected);
            errors++;
        }
    }

    if (errors == 0) {
        printf("Vector addition successful! All %d elements verified.\n", N);
    }

    // Free memory
    hipFree(d_a);
    hipFree(d_b);
    hipFree(d_c);
    free(h_a);
    free(h_b);
    free(h_c);

    return errors == 0 ? 0 : 1;
}
```

### Compile and run

Compile with `hipcc`, the HIP compiler driver included with ROCm:

```bash
hipcc vector_add.cpp -o vector_add
./vector_add
```

Expected output:

```text
Vector addition successful! All 1024 elements verified.
```

<Note>
  `hipcc` is a compiler driver that wraps Clang with the appropriate HIP include paths and library flags. It targets your installed AMD GPU architecture automatically. You can pass standard C++ flags such as `-O3` and `-std=c++17`.
</Note>

## Troubleshooting

**GPU not detected by `rocminfo`**

- Confirm the `amdgpu` kernel module is loaded: `lsmod | grep amdgpu`
- Check that your user is in the `render` and `video` groups: `groups $USER`
- Reboot after installing ROCm if the kernel module was not previously loaded

**`hipcc` not found**

- Ensure `/opt/rocm/bin` is in your `PATH`: `echo $PATH`
- Source your shell config: `source ~/.bashrc`

**Permission denied on GPU device**

- Add your user to the `render` group: `sudo usermod -aG render $USER`
- Log out and back in for the change to take effect

For more detailed troubleshooting, see the ROCm installation FAQ in the [compatibility matrix](/reference/compatibility-matrix) documentation.
